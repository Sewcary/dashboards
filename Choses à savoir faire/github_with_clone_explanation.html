Gérer ses fichiers avec GitHub - Méthode PACER

P - Procédural : Instructions pour GitHub

    • - Si tu n’as pas encore de compte, crée-en un sur GitHub.

    - Crée un nouveau dépôt dans GitHub, qui servira de stockage pour ton projet. Choisis de le rendre privé ou public selon tes besoins.

    • Sur ton ordinateur, navigue vers le répertoire que tu souhaites synchroniser avec GitHub. Ouvre un terminal et exécute les commandes suivantes :

    git init
    git remote add origin https://github.com/votre-nom-utilisateur/nom-du-depot.git

    • Ajoute tes fichiers au dépôt Git avec ces commandes :

    git add .
    git commit -m "Premier commit"
    git push origin master

Résolution des fichiers manquants sur GitHub

Très bien, résolvons ensemble cette chaîne de pensée pour comprendre pourquoi tu ne vois que le fichier README.md dans ton dépôt GitHub.

1. Vérification de l'initialisation du projet Git

Il semble que le dépôt a été bien initialisé avec git init, car tu vois le fichier README.md sur GitHub. Cependant, il serait utile de vérifier s'il y a d'autres fichiers dans ton répertoire local qui n'ont pas été ajoutés ou commis.

2. Ajout et validation des autres fichiers

Si tu n'as ajouté que le fichier README.md, les autres fichiers ne seront pas visibles dans ton dépôt GitHub. Pour confirmer cela, exécute git status dans ton terminal pour voir s'il y a d'autres fichiers non suivis (untracked files) ou modifiés dans ton répertoire.

3. Ajout des fichiers manquants

Si tu vois des fichiers non suivis ou modifiés, utilise git add . pour ajouter tous les fichiers, puis fais un nouveau commit avec git commit -m "Ajout des fichiers manquants".

4. Pousser les nouveaux commits vers GitHub

Après avoir ajouté et validé les fichiers manquants, pousse-les vers ton dépôt GitHub avec git push origin main.

Essaie ces étapes et dis-moi si tu vois les autres fichiers après avoir effectué ces actions.

Cloner le dépôt sur un autre PC

Pour cloner ton dépôt GitHub sur un autre PC, voici les étapes à suivre :

1. Ouvrir un terminal ou invite de commandes

Sur le PC cible, ouvre un terminal (ou l'invite de commandes sur Windows) pour exécuter les commandes Git.

2. Cloner le dépôt

Exécute la commande suivante en remplaçant votre-nom-utilisateur et nom-du-depot par ton nom d'utilisateur et le nom de ton dépôt :

git clone https://github.com/votre-nom-utilisateur/nom-du-depot.git

Cette commande va créer un répertoire local sur le PC cible avec tous les fichiers du dépôt GitHub.

3. Naviguer dans le répertoire cloné

Une fois le clonage terminé, tu peux te déplacer dans le répertoire nouvellement créé en utilisant la commande suivante :

cd nom-du-depot

4. Synchronisation future

Pour récupérer les nouvelles modifications du dépôt à l'avenir, tu peux utiliser la commande git pull :

git pull origin main
