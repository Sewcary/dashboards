Gérer ses fichiers avec GitHub - Méthode PACER

P - Procédural : Instructions pour GitHub

- Si tu n’as pas encore de compte, crée-en un sur GitHub.

- Crée un nouveau dépôt dans GitHub, qui servira de stockage pour ton projet. Tu peux choisir de le rendre privé ou public selon tes besoins.

Sur ton ordinateur, navigue vers le répertoire que tu souhaites synchroniser avec GitHub. Ouvre un terminal et exécute les commandes suivantes :

git init
git remote add origin https://github.com/votre-nom-utilisateur/nom-du-depot.git

Ajoute tes fichiers au dépôt Git avec ces commandes :

git add .
git commit -m "Premier commit"
git push origin master

A - Analogous : Comparaison avec d'autres systèmes

GitHub offre des avantages comparables à Dropbox ou Google Drive, mais se distingue par la gestion des versions et la collaboration en temps réel.

C - Conceptuel : Comprendre la gestion des versions

Les branches permettent de travailler sur plusieurs versions d’un projet. Les commits enregistrent chaque modification. GitHub facilite la collaboration en fusionnant les branches.

E - Evidence : Preuves de l'efficacité

Selon une enquête menée par Stack Overflow, 80 % des développeurs utilisent GitHub pour la gestion de projets collaboratifs, prouvant son efficacité.

R - Référence : Commandes Git essentielles

  • Ajouter des fichiers : 
    git add .
  • Valider les modifications : 
    git commit -m "Message de commit"
  • Envoyer les changements : 
    git push origin master
  • Récupérer les dernières modifications : 
    git pull
