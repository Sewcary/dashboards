Gérer ses fichiers avec GitHub

1. Créer un dépôt GitHub

- Si tu n’as pas encore de compte, crée-en un sur GitHub.

- Crée un nouveau dépôt dans GitHub qui servira de stockage pour ton projet. Tu peux choisir de le rendre privé ou public selon tes besoins.

2. Initialiser Git en local

Sur ton ordinateur, navigue vers le répertoire que tu souhaites synchroniser avec GitHub. Ouvre un terminal ou invite de commandes et exécute les commandes suivantes :

git init
git remote add origin https://github.com/votre-nom-utilisateur/nom-du-depot.git

3. Ajouter et envoyer des fichiers

Ajoute tes fichiers au dépôt Git avec ces commandes :

git add .
git commit -m "Premier commit"
git push origin master

Tous les fichiers de ton répertoire local seront envoyés à GitHub.

4. Synchronisation sur un autre PC

Sur un autre ordinateur, tu peux cloner le dépôt GitHub pour récupérer tous les fichiers :

git clone https://github.com/votre-nom-utilisateur/nom-du-depot.git

Cela créera un répertoire local avec tous tes fichiers sur cet appareil.

5. Modification et mise à jour

Lorsque tu modifies un fichier, tu peux utiliser les commandes suivantes pour ajouter et envoyer les changements sur GitHub :

git add .
git commit -m "Description des modifications"
git push origin master

Sur un autre ordinateur, utilise la commande suivante pour récupérer les dernières modifications effectuées :

git pull

6. Gestion des versions

GitHub permet également de gérer plusieurs versions d'un même fichier en utilisant des branches et en fusionnant les modifications. C’est une excellente option si plusieurs personnes travaillent sur le même projet.

7. Accès hors ligne

Les fichiers sont stockés localement et accessibles même hors ligne. Une fois que tu as une connexion Internet, tu peux synchroniser tes modifications avec GitHub en utilisant la commande suivante :

git push

8. Sécurité et confidentialité

Si tu veux garder le contrôle total sur qui peut accéder à tes fichiers, tu peux définir ton dépôt comme privé. Pour éviter de partager des informations sensibles (comme des mots de passe), ajoute un fichier .gitignore pour exclure certains fichiers de la synchronisation.

Résumé des commandes Git de base :

  • Ajouter des fichiers : 
    git add .
  • Valider les modifications : 
    git commit -m "Message de commit"
  • Envoyer les changements à GitHub : 
    git push origin master
  • Récupérer les dernières modifications : 
    git pull
