Gérer ses fichiers avec GitHub - Méthode PACER

P - Procédural : Instructions pour GitHub

Créez un nouveau dépôt dans GitHub, choisissez de le rendre privé ou public selon vos besoins.

Utilisez la commande suivante pour initialiser Git dans votre répertoire local :

git init
git add .
git commit -m "Premier commit"
git push origin master

A - Analogous : Comparaison avec d'autres systèmes

GitHub offre des avantages similaires aux outils comme Dropbox ou Google Drive, mais se distingue par la gestion des versions et la collaboration en temps réel.

C - Conceptuel : Comprendre la gestion des versions

Les branches permettent de travailler sur plusieurs versions d’un projet en parallèle. Les commits enregistrent chaque modification du projet.

E - Evidence : Preuves de l'efficacité

Selon une enquête menée par Stack Overflow, 80 % des développeurs utilisent GitHub pour la gestion de leurs projets, prouvant ainsi son efficacité.

R - Référence : Commandes Git à mémoriser

Voici quelques commandes essentielles à retenir :

git add .
git commit -m "Message de commit"
git push origin master
