Principes de Partitionnement dans Azure Synapse Analytics

Le partitionnement dans Azure Synapse Analytics consiste à diviser des données volumineuses en segments plus petits appelés partitions, afin d'améliorer les performances de requêtes et l'évolutivité des bases de données analytiques.

  • Le partitionnement permet de réduire les temps de réponse des requêtes en accédant à un sous-ensemble de données plutôt qu'à une table entière.

  • Les critères typiques incluent les dates, les régions géographiques, ou d'autres colonnes clés des tables de données. Ces colonnes sont souvent utilisées dans les filtres de requêtes.

Tâche Pratique : Implémenter une Stratégie de Partitionnement pour Azure Synapse

Dans cette tâche pratique, vous allez créer une table partitionnée, insérer des données, et optimiser les requêtes en fonction des partitions. L'objectif est de configurer un modèle performant dans Azure Synapse Analytics.

  • Créez une table partitionnée dans Azure Synapse en utilisant la colonne Date pour partitionner les données. Voici un exemple de requête SQL :

    CREATE TABLE Transactions
(
    DateTrans DATE,
    ID_Produit INT,
    Montant DECIMAL(18, 2),
    Région VARCHAR(50)
)
WITH (PARTITION (DateTrans RANGE RIGHT FOR VALUES ('2021-01-01', '2022-01-01')));

  • Insérez des données dans les partitions en fonction des dates. Utilisez des INSERT INTO avec des lots de données représentatifs.

  • Exécutez des requêtes en filtrant par partitions. Par exemple, effectuez des recherches sur une plage de dates spécifique pour optimiser les temps de réponse.

  • Utilisez Azure Monitor pour surveiller les performances des partitions et ajustez la stratégie si nécessaire (par exemple, augmentez le nombre de partitions).
