Aspects Conceptuels

Qu'est-ce que Transact-SQL (T-SQL) ?

  • Définition : Transact-SQL (T-SQL) est une extension du langage SQL standard, utilisée pour interagir avec les bases de données relationnelles dans SQL Server et Azure Synapse Analytics. Il permet d'exécuter des requêtes, de manipuler des données et de gérer des transactions complexes. T-SQL ajoute des fonctionnalités comme les boucles, les conditions (IF, CASE), et les blocs de transactions (BEGIN TRANSACTION, COMMIT).

  • T-SQL est essentiel dans Azure Synapse pour effectuer des opérations de manipulation et de transformation de données sur de grands ensembles de données. Il permet de charger, transformer et analyser les données en utilisant un langage bien connu par les professionnels SQL, tout en tirant parti des capacités d'échelle et de performance de Synapse Analytics.

Fonctionnalités clés de T-SQL

  • T-SQL permet d'exécuter des requêtes complexes avec des jointures, des sous-requêtes et des agrégations pour extraire des informations pertinentes de grandes tables de données.

  • Avec T-SQL, vous pouvez insérer, mettre à jour et supprimer des enregistrements dans des tables, ainsi que manipuler des données directement dans la base de données avec les instructions INSERT, UPDATE, DELETE.

  • Les blocs transactionnels (BEGIN TRANSACTION, COMMIT, ROLLBACK) vous permettent de gérer les transactions de manière sécurisée pour garantir l'intégrité des données lors de modifications complexes.

Atelier Pratique : Transformer des Données avec Transact-SQL dans Azure Synapse

Dans cet atelier, vous allez manipuler et transformer des données en utilisant des requêtes T-SQL dans Azure Synapse Analytics. Vous allez apprendre à exécuter des transformations complexes sur des tables et des vues, à filtrer des données, et à générer des résultats agrégés pour vos analyses.

  • Utilisez une requête T-SQL pour lire des données d'une table existante dans Azure Synapse et appliquez un filtre pour sélectionner des données spécifiques.

    • Accédez à votre espace de travail Azure Synapse et ouvrez un nouvel éditeur SQL.
    • Exécutez une requête T-SQL simple pour lire des données depuis une table : 
      SELECT *
FROM ventes
WHERE montant_vente > 500;
    • Ajoutez d'autres filtres pour raffiner les résultats : 
      SELECT *
FROM ventes
WHERE montant_vente > 500
AND date_vente > '2023-01-01';

  • Utilisez des fonctions d'agrégation comme SUM, COUNT et AVG pour obtenir des statistiques agrégées à partir des données.

    • Exécutez une requête qui calcule le total des ventes par produit : 
      SELECT produit_id, SUM(montant_vente) AS total_ventes
FROM ventes
GROUP BY produit_id;
    • Ajoutez une condition pour ne regrouper que les ventes supérieures à un certain seuil : 
      SELECT produit_id, SUM(montant_vente) AS total_ventes
FROM ventes
WHERE montant_vente > 500
GROUP BY produit_id;

  • Modifiez des enregistrements existants dans une table en utilisant l'instruction UPDATE pour corriger ou enrichir les données.

    • Exécutez une requête pour mettre à jour un champ dans une table : 
      UPDATE ventes
SET statut_vente = 'Validée'
WHERE montant_vente > 1000;
    • Vérifiez les enregistrements mis à jour en exécutant une requête SELECT sur la table modifiée.

  • Utilisez une transaction T-SQL pour vous assurer que toutes les modifications sont appliquées avec succès, ou qu'elles sont annulées en cas d'échec.

    • Commencez une transaction : 
      BEGIN TRANSACTION;
    • Mettez à jour plusieurs enregistrements dans une table : 
      UPDATE ventes
SET montant_vente = montant_vente * 1.1
WHERE date_vente > '2023-01-01';
    • Vérifiez les modifications : 
      SELECT * FROM ventes WHERE date_vente > '2023-01-01';
    • Si tout semble correct, appliquez les modifications : 
      COMMIT;
    • Si vous voulez annuler les changements, exécutez : 
      ROLLBACK;
